  1. 1.作品の実装に入る前に必ずスケッチを描いておく

    • グループ内での共通認識を持っておくため。

  2. 2.実装するのに必要なものを洗い出す

    • application, addon, device, ...etc

  3. 3.早い段階でパーツごとの技術検証をしておく

    • Kinect、画像認識、物理演算、など個々の機能が実装可能かを検証する。
    • 不可能な場合は別の方法を探す。

  4. 4.addonを使う場合はなるべくググらない

    • addonには複数のバージョンがあるものが多くありバージョンごとに仕様が違うため、ネット上に上がっている記事で扱っているaddonといま自分が使っているaddonのバージョンが一致しないことがある。
    • addonを使う際はまずofxXXX/exampleのsrcを見て、それでもわからなかったらググる。
    • ググるときは記事の日付やバージョンが同じかどうかを確認する。

  5. 5.技術検証ができたらパーツを組み合わせてデモをつくる

    • この段階ではシステムが動作していればOK。見た目はどうでもいい。

  6. 6.見た目を整える

    • プロジェクター、モニターを使用する場合は、最終的に表示される画面サイズでちゃんと表示されるようにする。

  7. 7.実機で動作検証、耐久テスト

    • なるべく展示と同じ状況、同じ時間でテストする。

  8. 8.管理、運用のしやすさを追求

    • キーアサインなどをapp内に表示しておき、マニュアルがなくても操作できるようにしておく。
    • 関係者以外でも操作できるくらいにしておくとなお良し。
    • 2の段階でsensorApp,renderAppなど機能別にアプリケーションを分割しておくと管理がしやすい。
      • ひとつに全部つめこんでしまうと一部の機能が落ちたときにアプリケーション全体が道連れになってしまう
      • デスクトップが見えるのだけは絶対に阻止したい
      • 分けておくと、途中でセンサーが落ちても表示側のアプリケーションには影響しない
